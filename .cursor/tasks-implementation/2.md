# Plan: Implement "findhim" Task (AI_Devs S01E02)

This plan outlines the steps to solve the **findhim** task by creating an automated agent that identifies which suspect from a provided list has been near a nuclear power plant, retrieves their access level, and submits the report to `/verify`.

## Goal

Identify the suspect who has been near a power plant, retrieve their security clearance level, determine the power plant code, and submit the result.

## Architecture

Follow the existing project structure under `src/tasks/2`:

- `types.ts` – TypeScript definitions for:
  - `Suspect` (from `.lessons/1/s01e01-answer.json`)
  - `Location` (latitude, longitude)
  - `PowerPlant` (from `findhim_locations.json`)
  - `AccessLevelResponse` (from `/api/accesslevel`)
  - `VerifyPayload` (structure for `/verify` endpoint)

- `tools.ts` – Implement utility functions:
  - `getPowerPlants(): Promise<PowerPlant[]>` – fetch plant list
  - `getSuspectLocations(name: string, surname: string): Promise<Location[]>` – fetch suspect locations
  - `getAccessLevel(name: string, surname: string, birthYear: number): Promise<AccessLevelResponse>`
  - `isNearPowerPlant(point: Location, plants: PowerPlant[]): string | null` – use Haversine formula to check proximity (e.g., <20 km) and return nearest plant code
  - `reportResult(payload: VerifyPayload): Promise<void>` – POST final answer to `/verify`

- `index.ts` – Main agent entry point:
  1. Load suspects from `.lessons/1/s01e01-answer.json`.
  2. Initialize OpenAI client with Function Calling definitions for each tool.
  3. Prompt the model to:
     - Retrieve the plant list.
     - Iterate suspects:
       1. Fetch locations.
       2. Check proximity to any plant.
       3. If close, fetch access level and prepare answer.
       4. Stop further iteration once a match is found.
     - Submit the report.

## Steps

1. **Define Types** (`src/tasks/2/types.ts`):
   - Declare interfaces for all data structures and API schemas.

2. **Implement Tools** (`src/tasks/2/tools.ts`):
   - Write fetch wrappers for each external API.
   - Implement the Haversine distance function.
   - Ensure robust error handling and early returns on failures.

3. **Build Agent Logic** (`src/tasks/2/index.ts`):
   - Set up OpenAI `ChatCompletion` with `functions` array describing tool schemas.
   - In the system prompt, outline the task and iteration logic.
   - In the user prompt, pass suspect list and instruct the model on the sequence.
   - Parse function calls from the model and dispatch to implemented tools.
   - Collect the final function call for reporting and execute `reportResult`.

4. **Testing & Verification**:
   - Run locally with a test API key.
   - Confirm the script outputs a flag `{FLG:*}`.

## Dependencies

- `openai` – for LLM interaction and Function Calling
- `dotenv` – for environment variables (API key)
- Native `fetch` (Node 18+) or `node-fetch` polyfill

## Risks & Edge Cases

- **No suspect found**: handle gracefully and log a clear message.
- **API errors/timeouts**: implement retries or fail-fast.
- **Large token usage**: instruct model to stop on first match.
- **Infinite loop in agent**: cap the loop at suspects.length.
